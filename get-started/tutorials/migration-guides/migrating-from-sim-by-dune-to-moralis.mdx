# Migrating from Sim by Dune to Moralis

<Info>
Sim (`sim.dune.com`) is being sunset on **August 1, 2026**. New signups were disabled on May 18, 2026. Any production workload backed by Sim needs a replacement read layer before the cutoff — this page maps the Sim API surface to Moralis endpoint-by-endpoint so the swap is a search-and-replace in most cases.
</Info>

Sim and Moralis cover the same wallet-centric problem space: real-time balances, activity, NFTs, DeFi positions, token metadata, and webhook subscriptions across 60+ EVM chains and Solana. Moralis is a drop-in target for the vast majority of Sim usage — the field names differ, but the resource shapes line up.

## Endpoint walkthroughs

### Wallet activity

**Sim**:

```bash
curl "https://api.sim.dune.com/v1/evm/activity/0xd8da6bf26964af9d7eed9e03e53415d37aa96045?chain_ids=1,8453" \
  -H "X-Sim-Api-Key: $SIM_API_KEY"
```

**Moralis**:

```bash
curl "https://deep-index.moralis.io/api/v2.2/wallets/0xd8da6bf26964af9d7eed9e03e53415d37aa96045/history?chain=eth" \
  -H "X-API-Key: $MORALIS_API_KEY"
```

Moralis returns a normalized, chronological feed of native transfers, ERC20 movements, NFT transfers, swaps, contract interactions, and approvals in a single call. See [Wallet History](/data-api/evm/wallet/wallet-history). For multi-chain rollups, call the endpoint per chain in parallel or use [Cross-Chain Token Balances](/data-api/universal/wallet/cross-chain-token-balances) for the balance view.

### Token balances

**Sim**:

```bash
curl "https://api.sim.dune.com/v1/evm/balances/0xd8da6bf26964af9d7eed9e03e53415d37aa96045?chain_ids=1" \
  -H "X-Sim-Api-Key: $SIM_API_KEY"
```

**Moralis**:

```bash
curl "https://deep-index.moralis.io/api/v2.2/wallets/0xd8da6bf26964af9d7eed9e03e53415d37aa96045/tokens?chain=eth" \
  -H "X-API-Key: $MORALIS_API_KEY"
```

The response includes balance, decimals, USD price, USD value, and token metadata (symbol, name, logo, verified-contract flag, possible-spam flag). See [Get Token Balances by Wallet](/data-api/evm/wallet/token-balances).

### NFTs / collectibles

```bash
  -H "X-Sim-Api-Key: $SIM_API_KEY"
```

**Moralis**:


Moralis returns ERC721 and ERC1155 holdings with normalized metadata, hosted media previews, and spam indicators. See [Get NFTs by Wallet](/data-api/evm/wallet/nft-balances).

### Webhooks → Streams

Sim webhooks deliver events for a watched address; Moralis Streams does the same and adds contract-event ABIs, smart-filter expressions, and internal-transaction visibility. Create a stream once and attach addresses to it dynamically:

```bash
# Create a stream that fires on any activity for a watched set of addresses
curl -X POST "https://api.moralis-streams.com/streams/evm" \
  -H "X-API-Key: $MORALIS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "webhookUrl": "https://your.app/webhooks/moralis",
    "description": "Wallet activity",
    "tag": "wallet-activity",
    "chainIds": ["0x1", "0x2105"],
    "includeNativeTxs": true,
    "includeContractLogs": true,
    "includeInternalTxs": false
  }'
```

Then attach addresses to the stream as your user base changes — no need to recreate the subscription. See the [Streams overview](/streams/overview) and [EVM Streams](/streams/evm-streams).

## Chain coverage

Sim covers 60+ EVM chains plus Solana. Moralis covers a comparable footprint across EVM, Solana, and Bitcoin. The full list is on [Supported Chains](/data-api/supported-chains) — if you're migrating, do a one-time diff against the chains you've configured in Sim and flag any gaps before cutover.

## Migration checklist

1. **Get an API key** — [Get your API key](/data-api/get-your-api-key).
2. **Map your call sites** — replace each Sim endpoint with the equivalent above. Field names differ; most teams write a thin response adapter rather than threading new shapes through the codebase.
3. **Move webhooks to Streams** — create one Stream per event class (activity, ERC20 transfers, contract events) and attach addresses dynamically instead of one webhook per address.
4. **Verify chain coverage** — diff your Sim chain configuration against [Supported Chains](/data-api/supported-chains).
5. **Cut over before August 1, 2026** — Sim's API stops serving on that date.

## What you gain beyond a like-for-like swap

Sim's surface maps cleanly onto Moralis, but Moralis also exposes endpoints that don't have a Sim counterpart and that are commonly used by teams once they migrate:

- **Wallet PnL** — realized and unrealized PnL per wallet with token-level breakdown. See [Wallet PnL Summary](/data-api/evm/wallet/wallet-pnl-summary) and [Wallet PnL by Token](/data-api/evm/wallet/wallet-pnl).
- **Wallet Stats & Insights** — first/last activity, chain-level activity rollups, and behavioral signals. See [Wallet Stats](/data-api/evm/wallet/wallet-stats) and [Wallet Insights](/data-api/evm/wallet/wallet-insights).
- **Token approvals** — open ERC20 approvals for a wallet, used widely in revoke flows. See [Wallet Approvals](/data-api/evm/wallet/approvals).
- **Token discovery** — [Trending Tokens](/data-api/universal/token/trending-tokens), [Top Gainers](/data-api/universal/token/top-gainers), [Top Losers](/data-api/universal/token/top-losers), and [Filtered Tokens](/data-api/universal/token/filtered-tokens).
- **OHLCV & pair stats** — candle data and DEX pair analytics. See [Get OHLCV by Pair Address](/data-api/evm/token/swaps/pair-stats).
- **Cross-chain balances in one call** — see [Cross-Chain Token Balances](/data-api/universal/wallet/cross-chain-token-balances).
